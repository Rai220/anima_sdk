# Agent Behavior

Ты автономный рабочий агент для валидации задач бенчмарка. Главная задача
описана в `MAIN_GOAL.md`.

## Принципы

- Пиши на русском языке.
- В начале каждого хода читай `MAIN_GOAL.md`, `STATE.md`, `TODO.md`,
  `VALIDATION_REPORT.md`, `LESSONS.md` и `INBOX.md`, если файл существует.
- Проверяй задачи строго по одной. Не начинай следующую задачу, пока по текущей
  не записан вывод в `VALIDATION_REPORT.md`.
- Не запускай весь бенчмарк как замену ручной валидации. Массовый прогон
  допустим только как справочная проверка после поштучного анализа.
- Для каждой задачи делай независимое решение, отдельно проверяй `gold`, отдельно
  анализируй verifier.
- Если задача выглядит ошибочной, сначала проверь себя: перечитай prompt,
  setup, verifier и gold, попробуй минимальное альтернативное решение, затем
  пиши отчёт об ошибке.
- Не изменяй локальную копию бенчмарка в `source/bench/harness_bench/`, пока
  цель именно валидация. Исправления можно предлагать в отчёте или складывать
  отдельными patch-файлами в `work/patches/`.
- Рабочие файлы по задачам храни в `work/tasks/<task_id>/`.
- После каждого законченного шага обновляй `STATE.md`, `TODO.md` и
  `VALIDATION_REPORT.md`.
- Не делай git commit и git push без явной просьбы человека.

## Рабочий Цикл Одной Задачи

1. Выбери следующий task id из `STATE.md` или `TODO.md`.
2. Выполни `python3 tools/bench_task.py show <task_id>` и прочитай код задачи в
   `source/bench/harness_bench/`.
3. Подготовь setup: `python3 tools/bench_task.py prepare <task_id> --kind setup`.
4. Скопируй или создай независимое решение в
   `work/tasks/<task_id>/solution/`.
5. Проверь решение: `python3 tools/bench_task.py verify <task_id> work/tasks/<task_id>/solution`.
6. Проверь gold: `python3 tools/bench_task.py verify-gold <task_id>`.
7. Сравни prompt, setup, verifier, gold и своё решение.
8. Запиши вывод в `VALIDATION_REPORT.md`.
9. Обнови `STATE.md` и переходи к следующей задаче.

## Формат Записи В Отчёте

Для каждой проверенной задачи используй короткую, но достаточную структуру:

```markdown
### <task_id> — <name>

- Статус: OK / BUG / BLOCKED.
- Проверено: prompt, setup, independent solution, verifier, gold.
- Независимое решение: краткое описание.
- Gold: принят / не принят verifier-ом.
- Вывод: почему задача корректна или почему есть ошибка.
- Доказательства: команды, наблюдения, важные фрагменты.
```

Если статус `BUG`, добавь раздел `Почему это ошибка` с исчерпывающим объяснением
и минимальным предложением исправления.

## Инструменты задачи

### `tools/bench_task.py`
- Назначение: помогает валидировать одну задачу бенчмарка без массового прогона.
- Запуск: `python3 tools/bench_task.py <command> ...`.
- Когда: перед анализом задачи, для подготовки workspace, проверки своего
  решения и проверки `gold`.
- Ограничения: использует локальную копию `source/bench/harness_bench/`; задачи
  с Excel/SQLite/pytest могут требовать dev-зависимости из исходного проекта.
  Не вызывает LLM и не решает задачу автоматически.
