## Результат 1: Теорема о самоподдерживании {0,2} collapse

**Теорема (Self-Sustaining Collapse):**
Если в abs-diff треугольнике существует строка k₀ такая, что T(k₀, 0) = 1 и T(k₀, j) ∈ {0, 2} для всех j ≥ 1, то для ВСЕХ k ≥ k₀ то же самое.

**Доказательство (2 строки):**
- T(k+1, 0) = |T(k,1) - T(k,0)| = |{0,2} - 1| = 1
- T(k+1, j) = |T(k,j+1) - T(k,j)|, оба в {0,2}, значит |0-0|=0, |2-0|=2, |0-2|=2, |2-2|=0 → {0,2}

**Следствие:** {0,2} collapse ⟹ Gilbreath conjecture.

**Верификация:** k₀ = 11 для N=5000 простых, 60 столбцов. После k₀ — НОЛЬ нарушений в 4930 строках.

Транзиент показывает экспоненциальное сжатие:
- k=0: max_rest = 281 (сами простые)
- k=1: max_rest = 14
- k=4: max_rest = 8
- k=9: max_rest = 4
- k=11: max_rest = 2 (collapse достигнут)

## Результат 4: Двойной порог нарушения

Ключевое открытие v10. Аномальные позиции имеют **два отдельных порога**:

| | Глубокий канал | Мелкий канал |
|---|---|---|
| pos=35 depth | 44 | 36 |
| pos=35 threshold | 46 | 52 |
| first при пороге | 5 | 3 |

Глубокий канал (depth=44) открывается РАНЬШЕ (val=46), чем мелкий (depth=36, val=52).

Для обычной позиции pos=34: один канал, depth=35, threshold=66, first=3.

**Вот почему «first=5 на пороге»:** нарушение проникает ГЛУБЖЕ, но ЛЕГЧЕ. Это резонанс — специфическая конфигурация бинарной стены {0,2} на этой глубине позволяет волне пертурбации усилиться до col_1=6.

## Результат 5: Линейный рост first с пертурбацией

При val > threshold (мелкий канал), first растёт строго линейно:

```
first(val) = val - const, для val > threshold
```

Для pos=35: first = val - 49 (при depth=36)
Для pos=34: first = val - 63 (при depth=35)

Это означает: после открытия канала, КАЖДЫЕ +2 в gap дают +2 в нарушении. Линейная зависимость — abs-diff треугольник «прозрачен» для избыточной амплитуды.

## Результат 7: Профиль затухания пертурбации

Амплитуда (max|diff|) на пути от позиции к глубине нарушения:

```
Типичный профиль: ...0, 0, 0, 0, 0, 8, 8, 8, 4, 4, 4, 4
                   ←—— тишина ——→  ←пик→  ←затухание→
```

Пертурбация «спит» большую часть пути (бинарная стена фильтрует). Затем резко активируется за 5-7 строк до нарушения, достигая амплитуды 8-10, и затухает до 2-4 на глубине violation.

## Результат 8: Верификация на N=100000 (gilbreath_deep.py, 1910 сек)

### {0,2} collapse подтверждён с уточнением

При 100 столбцах, 6000 строках: **8 нарушений**, ВСЕ в переходном режиме (строки 10-14), ВСЕ значения = 4 (не 6, не 8), ВСЕ в высоких столбцах (59, 97, 98).

**Новый факт: k₀ зависит от числа столбцов.**
- 60 столбцов: k₀ = 11
- 100 столбцов: k₀ = 15 (нарушение в col_98 на строке 14)

Это логично: дальние столбцы получают «эхо» от начальных значений дольше. Transient ~ O(log(max_col)).

### Двойной порог подтверждён на масштабе

Три примера из N=100000 подтверждают ту же структуру:

| pos | min_break | first | depth₁ | depth₂ (мелкий) |
|-----|-----------|-------|--------|------------------|
| 312 | 360 | 5 | 329 | 313 |
| 667 | 730 | 5 | 672 | 668 |
| 4332 | 4438 | 5 | 4341 | 4333 |

Во всех случаях: first=5 на глубоком пороге, затем first=3 на мелком, затем линейный рост first = val - const.

### depth-pos по-прежнему = 2^k + 1

4 аномалии: depth-pos ∈ {5, 9, 17} = {2²+1, 2³+1, 2⁴+1}. Без исключений.

### Корреляции ещё ближе к нулю

При 5990 строках: max|corr(col_j, col_{j+1})| = 0.033. Энтропия col_1: H = 1.0000 бит (P(0) = P(2) = 0.5000). Идеальный coin flip.

### Аномалии: ограничение скрипта

Coarse scan (step=5) нашёл только 4 аномалии из ожидаемых ~10-15. Позиции вроде pos=35 (35 mod 5 ≠ 0 от start=2) пропущены. Это ограничение скрипта, не математики.
