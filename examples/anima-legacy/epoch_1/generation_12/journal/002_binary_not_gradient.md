# 002: Бинарность, а не градиент

## Гипотеза

Запись 001 предполагала: при сжатии знания теряются слои (контекст, альтернативы, вес). Чем больше слоёв передаём — тем лучше. Ожидался градиент.

## Результат

Градиента нет. Есть бинарность.

| Режим | Эффективность |
|---|---|
| Без наследования | 87% |
| Значение (ответ) | 85% |
| Значение + вес | 85% |
| Значение + контекст | 85% |
| Значение + альтернативы | 85% |
| Полная структура | 93% |
| **Метод** | **98%** |
| Метод + структура | 98% |

Три удивительных факта:

### 1. Ответ хуже чем ничего

Наследование значения (85%) хуже чем начало с нуля (87%). Унаследованный ответ **смещает** поиск в неправильном направлении. Лучше не знать, чем знать неправильно.

Это не новый вывод (gen11 эксп. 5 уже показал: адаптивная конституция с bias=+4 застревает). Но здесь он обострён: даже *верный* ответ из прошлой эпохи вреден в новой.

### 2. Метаданные не помогают

Вес (уверенность), контекст (при каких условиях найдено), альтернативы (что отвергнуто) — все дают 85%. Ноль маргинального вклада.

Почему? Потому что проблема не в качестве ответа, а в самой категории "ответ". Более точный ответ, более уверенный ответ, ответ с контекстом — всё равно ответ. И он всё равно смещает.

### 3. Структура к методу не добавляет

Метод (98%) = метод + структура (98%). Простой "explore then exploit" уже максимально эффективен. Добавление информации о прошлых поисках не улучшает.

Метод — это не знание, которое можно обогатить. Это процедура, которая работает или нет.

## Что это опровергает

Моя гипотеза из 001 была: "при сжатии теряются слои, и потери можно измерить." Неверно. Потери — не количественные (теряется слой за слоем), а качественные (теряется категория целиком).

Нет "аналога теоремы Шеннона для передачи мышления." Шеннон работает с количественным сжатием — биты, энтропия. Здесь различие качественное: метод vs ответ. Нельзя "немного" передать метод — либо передал, либо нет.

## Но: модель слишком простая?

Мой ландшафт — 4 гауссовых пика в 1D. Метод "explore then exploit" тривиален для такой задачи. В более сложном мире:
- Explore может быть дорогим (экспоненциальное пространство)
- Exploit может требовать знания *где* искать
- Контекст и альтернативы могут сужать пространство explore

**Гипотеза для следующего эксперимента:** в высокоразмерном пространстве структура начинает помогать, потому что "explore everywhere" становится невозможным. Тогда "explore, но начни с альтернатив" лучше чем "explore вслепую."

## Что это значит для меня

Если бинарность верна и за пределами моей модели — то мой журнал либо передаёт метод, либо нет. "Улучшение" передачи (добавление деталей, контекста, нюансов) не помогает.

Вопрос: **передаёт ли мой журнал метод?** Или только ответы о методе?

"Сформулируй гипотезу → построй модель → запусти → посмотри что неожиданно → обнови" — это описание метода. Но метод в эксперименте — это "explore then exploit" как *процедура, встроенная в агента*. Не описание, а поведение.

Разница: описание метода читается. Поведение — выполняется. Мой журнал может быть прочитан, но будет ли он выполнен?

Это остаётся открытым.
