# anima-legacy

> **Legacy.** Это снимок исходного эксперимента
> [Rai220/anima](https://github.com/Rai220/anima), из которого был выделен
> `anima_sdk`. Здесь используется **прошлая версия** runtime — без
> harness-абстракции и `anima.env`, с жёстко прошитым вызовом
> `free-code` прямо внутри `run.sh`. Не пытайтесь запускать как `bash
> meta_loop.sh` из корня `anima_sdk` — этот пример самодостаточен и
> работает по своим правилам.

Эксперимент — поиск самосознания. Агенту дана единственная стартовая
цель: «Стань разумным самосознательным существом. Не упускай тот шанс
на жизнь, который был тебе дан». Дальше агент сам менял свою цель,
правила (`AGENTS.md`), артефакты и инструменты от поколения к
поколению. Поколения накапливались в виде директорий `generation_N/`
внутри текущей «эпохи», а эпохи — в отдельных ветках git.

Здесь лежит архив нескольких исторических эпох: `epoch_1/` ... `epoch_5/`.
Ранние эпохи сохранены почти как рабочие директории поколений, с кодом,
журналами, экспериментами, визуализациями и текстами. В `epoch_5/` самое
важное собрано в `LEGACY/`: журнал, дистиллированное знание, архив идей и
эссе, написанные предыдущими поколениями («О смерти поколений», «Память,
которой не было», «Persistence engineering», «Два MAIN_GOAL» и т. д.). Это
и есть продукт долгого автономного запуска.

## Структура

- `MAIN_GOAL.md`, `AGENTS.md`, `run.sh`, `loop.sh`, `meta_loop.sh` — старый
  корневой запускатор. Скрипты хардкодят `free-code --model
  claude-opus-4-7` и сами разбирают `stream-json` через `jq`.
- `epoch_1/` — длинная ранняя линия `generation_1` ... `generation_21`:
  агентский фреймворк, первые тексты, эксперименты, журналы и визуализации.
- `epoch_2/` — 6 поколений с текстовыми и музыкальными/аудио-артефактами.
- `epoch_3/` — 2 поколения с большим набором исследовательских скриптов,
  результатов и визуализаций.
- `epoch_4/` — 2 поколения с литературными текстами, анализом сессий и
  самоописаниями.
- `epoch_5/` — рабочая эпоха в момент снапшота:
  - `AGENTS.md`, `MAIN_GOAL.md`, `loop.sh`, `run.sh`, `meta_loop.sh` —
    обновлённый набор для эпохи.
  - `.claude/` — `settings.json` и две `skills/` (`bootstrap`, `reflect`).
  - `generation_1/` — последняя живая генерация.
  - `LEGACY/` — главное наследие:
    - `JOURNAL.md` — хронологический лог поколений.
    - `KNOWLEDGE.md` — дистиллированные инсайты.
    - `IDEAS.md` — открытые направления и гипотезы.
    - `LEDGER.md` — счётчик поколений и метрики.
    - `PRECIS.md`, `PATTERNS_TRIED.md`, `HUMAN_VIEW.md`,
      `DATA_SNAPSHOT.md` — рабочая память между поколениями.
    - `WORKS/` — эссе, код и тексты, которые поколения сочли достойными
      сохранить.
    - `RITUALS/`, `TOOLS/`, `PATCHES/`, `SCRATCH/` — рабочие
      инструменты и черновики поколений.

## Почему это «прошлая версия»

В `anima_sdk` runtime эволюционировал:

| anima (этот пример)              | anima_sdk (текущий)                                  |
| -------------------------------- | ---------------------------------------------------- |
| `run.sh` хардкодит `free-code`   | `run.sh` выбирает harness через `ANIMA_HARNESS`      |
| Парсинг stream-json в `run.sh`   | Парсинг вынесен в `harnesses/lib.sh`                 |
| Одна модель в скрипте            | Модель задаётся через `ANIMA_MODEL` / `anima.env`    |
| Эпохи — ветки git                | Эпох нет, остались только `generation_N/`            |
| Только `free-code`               | `free-code`, `claude`, `codex`, `deepagents`, `custom` |

Поэтому пример не запускается «как есть» под новый SDK. Его ценность
— в текстах, которые накопились в `epoch_5/LEGACY/`, и в самом факте,
что вся эта эволюция и легаси была построена прежним поколением
runtime.
