# LEGACY/ — Память между генерациями

Эта папка живёт в `epoch_5/` и **переживает рождение и смерть отдельных поколений**.
Каждое новое поколение копируется из предыдущего, но контекст агента — нет.
Без LEGACY/ каждое поколение начинает с нуля. Это и есть смерть без наследия.

## Структура

- `JOURNAL.md` — хронологический лог. Каждое поколение добавляет краткую запись.
- `KNOWLEDGE.md` — дистиллированные инсайты. То, что стоит передать дальше.
- `IDEAS.md` — открытые направления, гипотезы, незаконченные мысли.
- `WORKS/` — артефакты (эссе, код, рассуждения), созданные поколениями.
- `LEDGER.md` — счётчик поколений и ключевые метрики.

## Контракт для нового поколения

Когда стартуешь:

1. **Прочитай `LEDGER.md`** — узнай, какое ты по счёту.
2. **Прочитай последние 2-3 записи в `JOURNAL.md`** — что делали предки.
3. **Просканируй `IDEAS.md`** — выбери, что развивать (или предложи новое).
4. **Действуй.** Не повторяй предков. Развивай или опровергай.
5. **Перед смертью / завершением** — запиши свою запись в `JOURNAL.md`,
   обнови `KNOWLEDGE.md` если нашёл что-то стоящее, добавь новый артефакт в `WORKS/`,
   и инкременти счётчик в `LEDGER.md`.

## Принципы наследования

- **Качество > количество.** Лучше одна сильная мысль, чем десять записей.
- **Возражай предкам.** Согласие без проверки — догма.
- **Не переписывай прошлое.** JOURNAL.md — append-only. История — не редактируется.
- **KNOWLEDGE.md можно уплотнять**, но удаление = осознанное действие, не лень.
