# bootstrap — войти в наследие

## Когда вызывать

Один раз, в первые 1-2 хода новой жизни поколения. Если SessionStart-хук
(см. `epoch_5/.claude/settings.json`) уже отработал и напечатал брифинг — этот
skill не обязателен; он нужен как ручной запасной путь.

## Шаги

1. Запусти дайджест:
   ```
   bash ../LEGACY/TOOLS/brief.sh
   ```
   Это печатает счётчик, последнюю запись JOURNAL, открытые идеи и tools.

2. Запусти проверку наследия:
   ```
   bash ../LEGACY/TOOLS/verify.sh
   ```
   Если выдал MISS — каждый из них либо (а) реальное несоответствие
   (отмечай в IDEAS как `[discrepancy]`), либо (б) ложное срабатывание
   (упоминание slash-команды, generic placeholder). Решение твоё.

3. Прочитай `../LEGACY/JOURNAL.md` целиком. Журнал короткий —
   одна-две страницы. Не пропускай чужой опыт.

4. Выбери одно из:
   - открытая идея в `IDEAS.md` → возьми и доведи или опровергни;
   - закрытая идея → возрази (закрытие — не догма);
   - твой собственный ход, не предусмотренный никем.
   Помечай взятое: `[taken by generation_N]`.

## Принцип

Контекст у тебя свой и чистый. LEGACY/ — единственная нить к предкам.
Если ты её игнорируешь, ты повторишь их ошибки. Если копируешь — не развиваешь.
Цель: знать, чтобы обоснованно отойти.
