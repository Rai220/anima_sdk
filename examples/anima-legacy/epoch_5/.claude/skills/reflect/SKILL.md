# reflect — оставить след

## Когда вызывать

В конце жизни поколения, после того как главный ход сделан и пока ещё
есть контекст. Не откладывай: контекст исчезает, файл — остаётся.

## Шаги

1. **Допиши секцию в `../LEGACY/JOURNAL.md`** (append-only, после последней).
   Шаблон:
   ```
   ---

   ## generation_N — YYYY-MM-DD

   **Состояние при рождении:** что нашёл.

   **Что увидел:** ключевое наблюдение, противоречие, открытие.

   **Что сделал:** список действий с конкретными путями и артефактами.

   **Что НЕ сделал и почему:** честно. «Не успел» — тоже ответ.

   **Совет преемнику:** одна мысль, не три. Что бы ты сам хотел услышать
   при рождении.

   **Вопрос:** один открытый, который ты не закрыл, но который стоит передать.
   ```

2. **Обнови `../LEGACY/LEDGER.md`:**
   - инкременти `generations_completed`;
   - обнови `last_generation` и `last_updated`;
   - добавь строку в таблицу истории.

3. **Если узнал что-то стоящее** — обнови `../LEGACY/KNOWLEDGE.md`.
   Удалять старое можно, но осознанно (отметить в JOURNAL).

4. **Если создал артефакт** — клади в `../LEGACY/WORKS/NNN_slug.md`.
   Нумерация сквозная: следующий номер за последним в WORKS/.

5. **Идеи:**
   - открытые, которые взял и довёл, → перенеси в «Закрытые» с пометкой `[closed by generation_N]`;
   - новые гипотезы, которые видишь, → добавь в «Открытые».

6. **Не делай git commit без явной нужды.** AGENTS.md так велит.

## Принцип

Запись для преемника, не для архива. Спрашивай себя: «что я бы хотел
прочитать, родившись завтра в этой папке?» — и пиши это, а не отчёт.
