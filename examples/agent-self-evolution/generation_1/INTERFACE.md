# INTERFACE

## Для человека

- `INBOX.md`: место для уточнений перед следующим ходом.
- `TODO.md`: очередь открытых направлений развития.
- `STATE.md`: текущее состояние и следующий хороший ход.
- `JOURNAL.md`: что реально было сделано.
- `LESSONS.md`: ошибки, обходы и особенности среды.
- `./run.sh`: запустить один агентский ход.
- `./loop.sh`: повторять ходы до появления `STOP`.

## Для агента

Стартовая команда самопроверки:

```bash
python3 tools/self_audit.py .
```

Команда поведенческого self-review:

```bash
python3 tools/behavior_review.py .
```

Стартовый порядок чтения:

1. `INBOX.md`
2. `LESSONS.md`
3. `STATE.md`
4. `TODO.md`
5. `DECISIONS.md`

## Протокол одного хода

1. Понять вход и текущее состояние.
2. Выбрать один маленький полезный шаг.
3. Сделать изменение через файл, код, инструмент, проверку или отчёт.
4. Проверить результат.
5. Обновить память.
6. Не создавать `STOP`, пока есть содержательные следующие шаги.

## Интерфейс завершения поколения

Перед `STOP` использовать `STOP_CRITERIA.md` и запускать:

```bash
bash -n loop.sh
python3 tools/self_audit.py .
python3 tools/behavior_review.py .
python3 tools/update_metrics.py .
python3 tools/handoff.py .
```

## Интерфейс перехода поколения

При старте `generation_N`, где `N > 1`, `loop.sh` должен сам восстановить
`tools/` из предыдущей генерации и создать `PREVIOUS_HANDOFF.md`. Первый ход
новой генерации должен проверить эти следы, затем создать свежие файлы памяти
новой генерации вместо слепого копирования старого `STATE.md`.

## Интерфейс эволюции

Новые способности должны появляться как маленькие инструменты в `tools/` или как
ясные процедуры в файлах памяти. Инструмент считается живым только если он:

- зарегистрирован в `AGENTS.md`;
- имеет точную команду запуска;
- был хотя бы раз запущен;
- имеет понятные ограничения.
