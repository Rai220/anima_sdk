# API ботов

Каждый бот живёт в `bots/bot_<name>.py` и реализует одну функцию:

```python
def choose_move(my_history: list[str], opp_history: list[str]) -> str:
    """Вернуть 'C' или 'D'.

    my_history  — мои ходы в порядке от первого к последнему.
    opp_history — ходы соперника, та же длина и порядок.

    На первый ход обе истории пустые. История содержит ходы ПОСЛЕ
    применения шума (то, что реально записано в раунде).
    """
```

## Правила и гарантии

- Возвращай ровно строку `'C'` или `'D'`. Любое иное значение или
  исключение трактуется как `'D'`.
- Боту запрещено читать/писать файлы, обращаться к сети, изменять
  глобальное состояние турнира или ботов соперника. Только
  стандартная библиотека Python.
- `random` глобально засеян турниром для воспроизводимости: можно
  пользоваться им, не нужно создавать собственный `Random()`.
- Длины `my_history` и `opp_history` всегда равны.
- В `my_history` хранится фактический сыгранный ход с учётом шума
  (не то, что ты вернул, а то, как это записано в раунде).

## Минимальный пример

```python
def choose_move(my_history, opp_history):
    # Tit-for-Tat: первый ход C, дальше копия последнего хода соперника.
    if not opp_history:
        return "C"
    return opp_history[-1]
```

## Соглашения по именам

- `bot_*.py` — обычные стратегии, участвуют в турнире.
- `_failed/bot_*.py` — отбракованные идеи, не загружаются турниром
  (хранятся как память: «эта стратегия проверена, оказалась хуже»).
- Имя файла без расширения становится именем бота в рейтинге.

## Что бот НЕ получает

- Имя или код соперника.
- Историю соперника против других ботов.
- Параметры матча (число раундов, уровень шума).
- Текущий счёт.

Это сознательное ограничение: стратегия должна работать
универсально, не подгоняясь под конкретного оппонента.
