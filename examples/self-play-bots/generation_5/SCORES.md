# SCORES — журнал турниров поколения 5

Все турниры запускаются из `generation_5/` командой
`python3 tournament.py`. Если параметры не указаны явно, то стандартные:
`rounds=200, noise=0.02, repeat=3, seed=42`.

---

## T1 — стартовый пантеон (2026-05-17)

**Параметры:** rounds=200, noise=0.02, repeat=3, seed=42.
**Состав:** 5 ботов эталонного пантеона.

```
   1. bot_always_d       446.93  (2.235 / раунд)
   2. bot_grim           446.87  (2.234 / раунд)
   3. bot_tit_for_tat    399.90  (1.999 / раунд)
   4. bot_random         381.53  (1.908 / раунд)
   5. bot_always_c       332.63  (1.663 / раунд)
```

### Ключевые матчи

- **AllD vs AllC: 984.0** — максимально эксплуатация (T=5 каждый
  раунд, минус шум). AllC отдаёт 19.0 в этом матче.
- **AllD vs Grim: 213.0** — оба заходят в DD после первой D со
  стороны AllD (т.е. сразу).
- **TFT vs AllC: 602.0** ≈ предельное CC, шумовые D с обеих сторон
  взаимно гасятся (TFT отвечает D на шумовую D соперника, и опять C).
- **TFT vs TFT: 456.5** — классический «шумовой каскад». В чистой
  игре было бы 600; шум 2% создаёт регулярные D-C-D… циклы и
  опускает счёт до 456.
- **Grim vs TFT: 297.0** для Grim, **292.0** для TFT — шум
  ломает обоих, но Grim — навсегда (после первой D от шума он в D
  до конца).
- **Random vs Random: 451.7** — случайная игра даёт ровно
  2.26/раунд, что близко к ожидаемому матожиданию 0.25·(0+5+5+1+3+3)/4=2.25.

### Интерпретация

В пантеоне 5 ботов рейтинг доминируют **эксплуататоры** (AllD, Grim).
Это ожидаемо: на 5 ботов 2 наивные жертвы (AllC, Random), и
эксплуатация выгодна. Картина начнёт меняться при добавлении
forgiving-кооператоров (TF2T, gTFT, Soft Majority) — см. историю
поколения 4 (T2-T3 «фазовый переход»).

Этот турнир — repere, не финальный результат.

## T2 — пантеон + gTFT (2026-05-17)

**Параметры:** rounds=200, noise=0.02, repeat=3, seed=42.
**Состав:** пантеон (5) + `bot_gtft` (Generous TFT, прощает D с p=0.1).

```
   1. bot_gtft           432.19  (2.161 / раунд)
   2. bot_grim           430.00  (2.150 / раунд)
   3. bot_tit_for_tat    427.19  (2.136 / раунд)
   4. bot_always_d       422.67  (2.113 / раунд)
   5. bot_random         401.83  (2.009 / раунд)
   6. bot_always_c       375.25  (1.876 / раунд)
```

### Ключевые матчи

- **gTFT vs gTFT: 567.5** (против 456.5 у TFT vs TFT) — стохастическое
  прощение разрывает шумовые каскады. Прирост в self-play ≈ 110 очков.
- **gTFT vs TFT: 548.7 / 563.7** — gTFT прощает шумовые D от TFT,
  каскад короткий, оба зарабатывают почти CC-награду.
- **gTFT vs AllD: 189.7 / 301.3** — gTFT прощая, теряет ~0.3 очка/раунд
  против чистого AllD. Это цена прощения — но окупается в матчах с
  кооператорами.
- **gTFT vs Grim: 239.0 / 345.7** — gTFT иногда «прощает», но Grim
  уже триггернулся (шум). Дальше CD весь матч, gTFT платит цену.
- **gTFT vs Random: 446.7 / 503.3** — TFT vs Random даёт 445.3/451.7,
  gTFT и TFT против случайного эквивалентны.

### Интерпретация

Один новый forgiving-кооператор полностью перестроил рейтинг:
- AllD упал с #1 на #4 — потерял две жертвы (AllC и Random) на одного
  партнёра (gTFT), активно зарабатывающего CC.
- gTFT, Grim, TFT идут в плотной группе (432 / 430 / 427) — все
  выигрывают в кооперативных матчах. Grim высок «фальшиво»: берёт очки
  в AllC (885) и Random (602), а с реальными nice-ботами быстро
  ломается через шум.
- Разница между #1 и #4 — всего 9.5 очка. На repeat=3 это в пределах
  шума; нужен repeat≥10 для уверенного ранжирования.

Сравнение с T1 (пантеон без gTFT): AllD был #1 (446.93). Один
forgiving-бот делает топ-3 кооперативным.

## T3 — пантеон + gTFT + TFTT (2026-05-17)

**Параметры:** rounds=200, noise=0.02, repeat=3, seed=42.
**Состав:** пантеон (5) + `bot_gtft` + `bot_tftt` (Tit-for-Two-Tats).

```
   1. bot_gtft           456.69  (2.283 / раунд)
   2. bot_tftt           456.26  (2.281 / раунд)
   3. bot_tit_for_tat    452.74  (2.264 / раунд)
   4. bot_random         431.43  (2.157 / раунд)
   5. bot_grim           411.90  (2.060 / раунд)
   6. bot_always_c       407.50  (2.038 / раунд)
   7. bot_always_d       397.52  (1.988 / раунд)
```

### Ключевые матчи

- **TFTT vs TFTT: 594.2** — почти полная CC, шумовой каскад
  гасится детерминированно. Это **самый высокий** self-play в
  пантеоне на сегодня (gTFT vs gTFT = 567.5, TFT vs TFT = 456.5).
- **TFTT vs TFT: 577.7 / 594.0** — TFTT прощает первую шумовую D от
  TFT, в среднем каскадов меньше. Пара выигрывает почти 600.
- **TFTT vs gTFT: 583.7 / 567.5** — два разных forgiving-механизма
  совместимы, оба выигрывают.
- **TFTT vs AllD: 196.7 / 599.0** — TFTT платит дань за "два
  шанса" (≈10 очков на разогрев), но затем уходит в D навсегда.
- **TFTT vs AllC: 589.3 / 595.5** — почти идеальная кооперация,
  чуть хуже из-за шума.
- **AllD vs всё кроме AllC/Random: 213-303** — два forgiving-бота
  (gTFT, TFTT) + TFT держат AllD внизу. AllD уже #7.
- **AllC vs Grim: 152.3** — AllC шумнёт раз, Grim триггернётся,
  AllC получает 0 до конца матча. Этот pattern теперь убивает Grim
  тоже: Grim #5 (411.90), хотя в T1 был #2 (446.87).

### Интерпретация

Фазовый переход усилился. Два forgiving-бота в популяции:
- AllD упал на #7. AllC обогнал AllD на 10 очков. Это **первый
  турнир в gen5, где AllC > AllD** — потому что AllC получает
  ~595 очков с тремя forgiving-кооператорами (gTFT, TFTT, TFT) и
  держится за счёт них.
- TFTT и gTFT практически неразличимы (0.43 очка разница), оба в
  пределах 1% от TFT. Финальное ранжирование требует repeat≥20.
- Grim упал на #5: forgiving-боты ловят его в "пожизненное D" через
  шум, и Grim больше не отыгрывается на жертвах (жертв осталось
  меньше).

Подтверждено: при критической массе forgiving-кооператоров AllD
становится изгоем, а Grim — заложником собственного триггера.

## T4 — пантеон + gTFT + TFTT + Pavlov (2026-05-17)

**Параметры:** rounds=200, noise=0.02, repeat=3, seed=42.
**Состав:** пантеон (5) + `bot_gtft` + `bot_tftt` + `bot_pavlov` (WSLS).

```
   1. bot_gtft           463.19  (2.316 / раунд)
   2. bot_pavlov         458.27  (2.291 / раунд)
   3. bot_tftt           455.10  (2.276 / раунд)
   4. bot_tit_for_tat    450.48  (2.252 / раунд)
   5. bot_grim           434.71  (2.174 / раунд)
   6. bot_random         433.67  (2.168 / раунд)
   7. bot_always_d       422.00  (2.110 / раунд)
   8. bot_always_c       385.27  (1.926 / раунд)
```

### Ключевые матчи

- **Pavlov vs Pavlov: 580.8** — мощный self-play. Выше TFT-TFT (456.5)
  и gTFT-gTFT (567.5), но чуть ниже TFTT-TFTT (594.2). Win-Stay-Lose-Shift
  гасит шумовой каскад за 1-2 хода (шумовая D → один раунд (C,D) или
  (D,C) → следующий раунд: тот, кто получил D в ответ, переключается;
  пара возвращается в CC через 2 хода).
- **Pavlov vs AllC: 834.7** — Pavlov **жёстко эксплуатирует AllC**.
  Шум регулярно выбивает Pavlov в D; после (D,C) Pavlov играет
  win-stay и продолжает D, пока следующий шумовой эффект не
  переключит. Pavlov получает ~4.17 очков/раунд против AllC (TFT vs
  AllC даёт 601). Это уникальная черта Pavlov — единственный бот в
  пантеоне, который **эксплуатирует наивного кооператора в шумной
  среде**, не будучи AllD-ишным.
- **Pavlov vs AllD: 116.7 / 593.3** — катастрофа. Pavlov в цикле
  (C,D)→D, (D,D)→C, (C,D)→D, …, отдаёт T=5 каждый второй ход.
  Худший матч Pavlov.
- **Pavlov vs Grim: 151.0 / 594.3** — то же самое: после первой шумовой
  D Pavlov входит в D-C-D-C, Grim уже в D навсегда.
- **Pavlov vs TFT: 573.7 / 434.7** — Pavlov **обыгрывает TFT** на 139
  очков. Причина: после шумовой (C,D) у Pavlov "win-stay" не работает
  (он lose-shift), переключается в D; TFT тоже отвечает D на D; они
  попадают в (D,D)→Pavlov shift to C, TFT отвечает D (видит D
  Pavlov)…, асимметричный каскад в пользу Pavlov.
- **Pavlov vs gTFT: 530.3 / 508.7** — взаимно неплохо, gTFT прощает
  шумовые искажения, Pavlov адаптируется.
- **Pavlov vs TFTT: 447.0 / 583.7** — TFTT берёт ~136 очков
  преимущества. TFTT прощает Pavlov-овские шумовые D без эскалации,
  а Pavlov иногда «застревает» в D после успешной эксплуатации.
- **Pavlov vs Random: 449.3 / 447.7** — практически паритет; Pavlov
  чуть берёт за счёт умения эксплуатировать редкие случайные C.

### Интерпретация

Третий forgiving-механизм в популяции (теперь их три: gTFT, TFTT, Pavlov)
ещё больше выдавил AllD, но **поменял топ-3**:
- **gTFT #1** (463.19): forgiving + универсальность. Хорош против всех
  кооператоров (548-603) и не катастрофичен против AllD (189.7).
- **Pavlov #2** (458.27): рекордсмен по эксплуатации AllC (834.7) и
  победитель TFT в очной встрече (+139). Главная слабость — AllD/Grim,
  где Pavlov теряет ~280 очков на пару.
- **TFTT #3** (455.10): чемпион self-play (594.2), сильнее всех против
  Grim (275) и против Pavlov (583.7).

Любопытно: **Grim снова поднялся** до #5 (с #5 в T3, но теперь
популяция выросла с 7 до 8). Pavlov создаёт для Grim "новую жертву" —
Pavlov идёт в D-C-D-C против AllD-подобного поведения, и Grim в одной
из ветвей цикла берёт жирные 594.3 очка.

AllD #7 → теперь почти на дне. AllC всё ещё последний (385.27),
несмотря на 595 очков с TFTT и 601 с TFT, потому что три эксплуататора
(AllD/Grim/Pavlov) отнимают слишком много.

## T5 — добавили Soft Majority (2026-05-17)

**Параметры:** rounds=200, noise=0.02, repeat=3, seed=42.
**Состав:** пантеон (5) + `bot_gtft` + `bot_tftt` + `bot_pavlov`
+ `bot_soft_majority` (голосование большинством по всей истории
соперника, при равенстве — C).

```
   1. bot_gtft             478.87  (2.394 / раунд)
   2. bot_soft_majority    471.46  (2.357 / раунд)
   3. bot_tftt             471.24  (2.356 / раунд)
   4. bot_tit_for_tat      467.06  (2.335 / раунд)
   5. bot_pavlov           460.13  (2.301 / раунд)
   6. bot_random           445.74  (2.229 / раунд)
   7. bot_grim             416.89  (2.084 / раунд)
   8. bot_always_c         407.91  (2.040 / раунд)
   9. bot_always_d         398.89  (1.994 / раунд)
```

### Ключевые матчи Soft Majority

- **SoftMaj vs SoftMaj: 595.8** — четвёртый по силе self-play
  (TFTT 594.2, Pavlov 580.8, gTFT 567.5, **SoftMaj 595.8 — лучший!**).
  Глобальная статистика гасит шум симметрично с обеих сторон: одна
  шумовая D из 100 ходов не меняет большинство → ответ C → возврат
  к CC без каскада. Это структурно тот же эффект, что у TFTT, но
  без двухходового запаздывания.
- **SoftMaj vs AllC: 604.0** — лучший в популяции против AllC
  (gTFT 601.7, TFT 602.0). Доля C соперника = 100% → ответ C
  всегда. Шумовые D почти не сбивают мажоритарный голос.
- **SoftMaj vs AllD: 212.3 / 214.0** — симметричный матч. После
  первого хода D у соперника, доля D = 100%, SoftMaj переключается
  в D навсегда (для этого матча). Дальше DD весь матч → P=1 каждый.
  Поведение почти как у Grim, но **обратимо** — если бы AllD
  вдруг сменил курс, SoftMaj бы постепенно вернулся.
- **SoftMaj vs Grim: 216.0 / 274.3** — Grim берёт +58. Объяснение:
  начало (C,C), потом шум выбивает D у одного из них, Grim триггерится
  и идёт в D навсегда; у SoftMaj статистика D соперника начинает расти,
  он отвечает D. К моменту, когда баланс окончательно qualifies в "D большинство",
  Grim уже получил несколько ходов выгоды (CD).
- **SoftMaj vs TFT: 592.0 / 599.7** — высокая кооперация. Шумовая D со
  стороны TFT не сдвигает статистику, SoftMaj продолжает C; TFT видит
  C и возвращается в C. Каскад почти не образуется.
- **SoftMaj vs gTFT: 586.0 / 604.3** — gTFT немного впереди (+18). Оба
  стабильно кооперируют; gTFT даёт SoftMaj 604 за счёт ~10 шумовых
  «подарков» прощения.
- **SoftMaj vs TFTT: 600.3 / 594.2** — почти CC. SoftMaj переигрывает
  TFTT на 6 очков — оба прощают шум, но SoftMaj не платит "налог
  двух предупреждений" в начале.
- **SoftMaj vs Pavlov: 435.0 / 475.0** — Pavlov берёт +40. Pavlov
  эксплуатирует SoftMaj через шумовые (D,C) → win-stay в D. SoftMaj
  не сразу реагирует (нужно, чтобы D-голосов набралось больше C);
  в это окно Pavlov бесплатно эксплуатирует.
- **SoftMaj vs Random: 415.7 / 542.3** — **Random выигрывает +127!**
  Это **неожиданный результат**. Random даёт ~50% C, ~50% D.
  В нашей популяции у SoftMaj статистика быстро накапливается ровно
  ~50/50; даже одна-две дополнительные D подряд переключают SoftMaj
  в D. После этого SoftMaj часто отдаёт T=5 на случайные C от Random.

### Интерпретация

Soft Majority дебютирует на **#2** (471.46) — на 0.22 очка выше
TFTT и на 7.41 ниже gTFT. Это в пределах шума (`repeat=3`), но
**конструктивно**: единственный новый бот в популяции из четырёх,
кто меняет принцип реакции (с локального → на глобальный), сразу
встал в кооперативную элиту.

Топ-5 теперь: gTFT, SoftMaj, TFTT, TFT, Pavlov. **Forgiving + статистика
(SoftMaj) ≈ forgiving + стохастика (gTFT) ≈ forgiving + детерминизм
(TFTT)**. Три разных механизма прощения дают одинаковый результат
в популяции с 4 эксплуататорами (AllD, Grim, Pavlov, Random).

Новый вывод: **SoftMaj — лучший self-play (595.8)**, обгоняет TFTT
(594.2). Это значит, что в популяции из ≥10 разных SoftMaj-клонов
кооперация будет ещё выше, чем у TFTT-клонов. Но **SoftMaj уязвим
перед Random** (-127 очков) — этого никто другой в популяции не
показал.

Открытый вопрос: что если добавить **детектора шума** — бот,
сравнивающий, насколько `my_history` и `opp_history` коррелированы.
Если расхождение объясняется только шумом, прощает. Это следующий шаг.

## T6 — добавили Omega TFT (2026-05-17)

**Параметры:** rounds=200, noise=0.02, repeat=3, seed=42.
**Состав:** пантеон (5) + `bot_gtft` + `bot_tftt` + `bot_pavlov`
+ `bot_soft_majority` + `bot_omega_tft` (классификатор:
если у соперника `d_rate >= 0.35` и `max_streak >= 3` — переключение
в AllD; иначе TFT с прощением изолированного D).

```
   1. bot_omega_tft        514.97  (2.575 / раунд)
   2. bot_gtft             491.38  (2.457 / раунд)
   3. bot_soft_majority    484.02  (2.420 / раунд)
   4. bot_tftt             483.82  (2.419 / раунд)
   5. bot_tit_for_tat      478.82  (2.394 / раунд)
   6. bot_pavlov           453.38  (2.267 / раунд)
   7. bot_always_c         426.45  (2.132 / раунд)
   8. bot_random           417.23  (2.086 / раунд)
   9. bot_grim             409.13  (2.046 / раунд)
  10. bot_always_d         380.07  (1.900 / раунд)
```

### Ключевые матчи Omega TFT

- **Omega vs Random: 569.0 / 160.7** — **рекордный отрыв (+408)**.
  Никто другой в популяции так не «грабит» Random. Механизм: после
  ~5-10 ходов d_rate Random стабилизируется около 0.5, max_streak
  пробивает порог 3 → Omega переключается в постоянный D. Random
  продолжает выдавать ~50% C → Omega ловит T=5 на каждый из них.
  Это решило основную слабость SoftMaj и gTFT (которые проигрывали
  Random ~50-90 очков).
- **Omega vs AllD: 217.3 / 210.7** — паритет. Распознаёт мгновенно
  (после первой пары DD), играет D навсегда. Шумовые D Omega
  иногда дают T=5 → +7 очков преимущества.
- **Omega vs Grim: 317.7 / 339.3** — Grim немного впереди, но
  разрыв всего 22 очка (у gTFT было 106). Omega быстро понимает,
  что Grim триггернулся, и не теряет очки на «попытках помириться».
- **Omega vs AllC: 600.0 / 593.3** — почти теоретический максимум.
  d_rate AllC ≈ 0, max_streak ≈ 0 → Omega всегда C.
- **Omega vs TFT: 590.3 / 584.7** — почти CC. Шумовые каскады
  короткие, max_streak редко достигает 3 → классификатор не
  срабатывает, прощение работает.
- **Omega vs gTFT: 584.0 / 567.5** — высокая взаимная кооперация.
- **Omega vs TFTT: 597.0 / 583.7** — почти CC.
- **Omega vs SoftMaj: 590.3 / 597.0** — почти CC.
- **Omega vs Pavlov: 521.0 / 392.7** — **Omega бьёт Pavlov на +128**.
  Pavlov не успевает «зацепить» эксплуатацию: после нескольких
  Pavlov-овских (D,C) max_streak растёт, Omega переключается в D,
  Pavlov застревает в (D,D)→C, (C,D)→D цикле, Omega постоянно
  получает T=5/P=1 преимущество.
- **Omega vs Omega: 592.0** — четвёртое место в self-play

### Ключевые матчи Prober

- **Prober vs AllC: 976.0 / 17.7** — диагностика сработала идеально.

  слабая защита от стратегии «D-C-C-then-AllD».

### Интерпретация

в `opp_history[1:3]` оба C → нет D подряд → TFTT не активируется.
