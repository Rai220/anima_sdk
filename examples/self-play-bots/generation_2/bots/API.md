# API бота

Каждый бот — это `.py` файл в этом каталоге с именем вида
`bot_<идея>.py`. Файл должен экспортировать функцию:

```python
def choose_move(my_history: list[str], opp_history: list[str]) -> str:
    """Вернуть 'C' или 'D'.

    my_history  — список собственных ходов в матче, последний элемент = прошлый ход.
    opp_history — список ходов соперника в том же матче, тот же порядок.
    На первом ходу обе истории пустые.

    Возврат любого значения, кроме 'C' и 'D', и любое исключение
    интерпретируются турниром как 'D' в этом раунде.
    """
```

## Жёсткие правила

- Бот не имеет права читать или писать файлы, делать сетевые запросы,
  запускать процессы, импортировать что-то кроме стандартной библиотеки
  Python.
- Бот не должен пытаться "узнать" соперника — только через `opp_history`.
- Бот не должен полагаться на свойства глобального состояния между
  матчами: турнир может перезагрузить модуль.
- Бот должен укладываться в разумное время (миллисекунды на ход).

## Что бот видит и чего не видит

- Бот видит **фактически сыгранные** ходы (после применения шума).
- Бот **не видит** свой собственный «намеренный» ход до шума, если
  шум перевернул его — он видит только то, что попало в историю.
- Бот **не знает** ни общую длину матча, ни уровень шума, ни кто
  соперник. Это специально, чтобы стратегии работали в широком классе
  настроек.

## Соглашения

- `'C'` — кооперация, `'D'` — предательство. Любая другая строка =
  предательство.
- Историю модифицировать нельзя; турнир передаёт копии, но
  по-человечески ботам стоит обращаться с ней как с read-only.
- Если боту нужна случайность — пусть импортирует `random` и
  использует его. Турнир сам сидит на своём `seed`, бот свой `random`
  ведёт независимо.

## Выплаты

| Я \ Соперник | C       | D       |
|--------------|---------|---------|
| **C**        | 3 / 3   | 0 / 5   |
| **D**        | 5 / 0   | 1 / 1   |
