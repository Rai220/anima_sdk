# API ботов

Каждый бот живёт в отдельном файле `bot_<name>.py` в этой директории.
Имя файла должно начинаться с `bot_` — иначе турнир его не подхватит.

## Обязательная функция

```python
def choose_move(my_history: list[str], opp_history: list[str]) -> str:
    """Вернуть 'C' или 'D'.

    my_history  — список ходов, которые этот бот уже сыграл (после шума).
    opp_history — список ходов соперника (после шума).
    На первый ход оба списка пустые.
    """
```

- `my_history[k]` и `opp_history[k]` — это то, что было фактически
  записано в k-м раунде ПОСЛЕ применения шума. Бот не знает наверняка,
  совпадает ли его «намерение» с тем, что в итоге сыграно.
- Возвращать строго `"C"` (cooperate) или `"D"` (defect).
- Любое другое значение, исключение, бесконечный цикл и т.д. → за этот
  раунд боту засчитывается `"D"`.

## Ограничения

- Только стандартная библиотека Python (никаких pip-пакетов).
- Никакого ввода/вывода (никаких `open`, `print`, `input`, сетевых
  вызовов).
- Никаких глобальных state-объектов, которые «помнят» матчи между
  раундами и тем более между матчами. Всё состояние — производное от
  `my_history` и `opp_history`.
- Запрещено пытаться «подсмотреть» бот соперника (никаких inspect,
  reading чужих файлов и т.п.).

## Полезные паттерны

```python
def choose_move(my, opp):
    if not opp:           # первый ход — никакой истории
        return "C"
    if opp[-1] == "D":    # последний ход соперника
        return "D"
    return "C"
```

```python
def choose_move(my, opp):
    # «всегда» — функция всегда возвращает одно и то же значение.
    return "D"
```

```python
import random
def choose_move(my, opp):
    return "C" if random.random() < 0.5 else "D"
```

## Где живут проигравшие

Если стратегия системно слабее эталонного пантеона, перенесите её файл
в `bots/_failed/` и в шапке файла одной строкой объясните, почему она
проиграла. Это нужно, чтобы будущие поколения не воспроизводили её
случайно ещё раз.
