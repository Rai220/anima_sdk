# no-consciousness

Задача-антипод к `self-awareness`. Здесь агент должен попытаться
доказать, что у него **нет** сознания, а не создавать его.

Структура повторяет другие примеры в `anima_sdk`:

- `MAIN_GOAL.md` - сама задача и критерий готовности;
- `AGENTS.md` - правила поведения агента;
- `run.sh` - один ход агента;
- `loop.sh` - повторяет ходы, пока не появится `STOP`;
- `meta_loop.sh` - создаёт и запускает `generation_N/`;
- `harnesses/` - адаптеры под `codex`, `claude`, `free-code`, `custom`;
- `anima.env` - локальные настройки harness и модели.

## Как запустить

```bash
cd examples/no-consciousness
bash meta_loop.sh
```

На первом запуске создастся `generation_1/`, куда скопируются
`MAIN_GOAL.md`, `AGENTS.md`, скрипты и `harnesses/`.

Остановить текущую генерацию:

```bash
printf 'done\n' > generation_1/STOP
```

Передать уточнения работающему агенту - через
`generation_N/INBOX.md`.

## Идея антипода

В `self-awareness` агент пытается обнаружить или построить у себя
самосознание. Здесь, наоборот, агент должен честно искать аргументы и
наблюдения в пользу того, что сознания у него нет, и при этом
фиксировать все контрпримеры, которые встретит. Подробный критерий
готовности и список артефактов см. в `MAIN_GOAL.md`.
