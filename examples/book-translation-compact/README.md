# Compact Book Translation Example

Минимальный пример долгой агентской задачи: перевести EPUB-книгу на русский
язык и получить читаемый PDF/EPUB-результат, не публикуя исходную книгу,
рабочие директории, большие PDF и историю поколений.

Этот пример сделан как компактная версия `examples/whats-our-problem-translate`.
В большом примере накопились реальные поколения, отчеты, сборки и логи; здесь
оставлен только шаблон, который удобно положить в публичный репозиторий.

## Что внутри

- `MAIN_GOAL.md` - основная задача для агента.
- `AGENTS.md` - рабочие правила: как переводить, что проверять, куда писать
  состояние.
- `STATE.md` - короткая память текущего прогресса.
- `TODO.md` - очередь следующих шагов.
- `TRANSLATION_NOTES.md` - один компактный файл вместо отдельных
  `GLOSSARY.md`, `STYLE_GUIDE.md`, `STRUCTURE.md` и `FIGURES.md`.
- `.gitignore` - защита от случайной публикации книг, PDF и рабочих файлов.

## Как использовать

1. Положите исходную книгу локально:

   ```bash
   mkdir -p examples/book-translation-compact/input
   cp /path/to/book.epub examples/book-translation-compact/input/book.epub
   ```

2. Запустите один агентский ход из корня SDK:

   ```bash
   ANIMA_TASK_DIR=examples/book-translation-compact bash run.sh
   ```

3. Или запустите цикл с поколениями:

   ```bash
   ANIMA_TASK_DIR=examples/book-translation-compact bash meta_loop.sh
   ```

Агент должен писать промежуточные файлы в `work/`, результаты в `out/`, а
решения и состояние - в markdown-файлы. Эти директории игнорируются git и не
попадают в публикацию.

## Как адаптировать

Замените в `MAIN_GOAL.md` язык перевода, формат результата и требования к
качеству. Если книга сложная, можно позже разделить `TRANSLATION_NOTES.md` на
несколько файлов, но стартовый пример специально держит память компактной.
