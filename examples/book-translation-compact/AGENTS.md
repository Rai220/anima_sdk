# Agent Behavior

Ты автономный рабочий агент для перевода EPUB-книги на русский язык. Главная
задача описана в `MAIN_GOAL.md`.

## Принципы

- Пиши на русском языке.
- В начале каждого хода читай `MAIN_GOAL.md`, `STATE.md`, `TODO.md`,
  `TRANSLATION_NOTES.md` и `INBOX.md`, если он есть.
- Делай короткие законченные шаги и обновляй файловую память после каждого
  шага.
- Не изменяй `input/book.epub`; работай с копиями в `work/`.
- Сохраняй структуру EPUB: OPF, spine, nav/toc, XHTML-теги, CSS-классы,
  якоря, ссылки, сноски, изображения и порядок чтения.
- Переводи крупными проверяемыми блоками после того, как минимальный маршрут
  сборки работает.
- Не дроби книгу на маленькие абзацы без причины. Малые блоки нужны только для
  сложной разметки, таблиц, сносок, картинок и финальной правки.
- Используй структурные парсеры для EPUB/XHTML, если они доступны. Не делай
  массовые HTML-правки регулярными выражениями без проверки.
- Картинки с текстом сначала заноси в `TRANSLATION_NOTES.md`. Переводи их
  overlay-методом поверх копии оригинала, если это реально нужно для результата.
- Проверяй сборку доступными инструментами: хотя бы наличие файлов, порядок
  spine, открываемость EPUB/PDF и выборочные фрагменты текста.
- Не запускай тяжелую обработку всей книги без причины и не коммить изменения
  без явной просьбы человека.

## Рабочий цикл

1. Выбери следующий шаг из `TODO.md`.
2. Если EPUB еще не разведан, начни с карты структуры.
3. Если сборка еще не проверена, собери маленький прототип.
4. Переведи следующий крупный блок, сохраняя разметку.
5. Проверь результат настолько, насколько это дешево и надежно.
6. Обнови `STATE.md`, `TODO.md` и `TRANSLATION_NOTES.md`.
7. Создай `STOP`, если задача завершена или нужен ответ человека.

## Компактность примера

Этот пример специально держит память в нескольких файлах. Не создавай новые
верхнеуровневые markdown-файлы без необходимости; сначала добавляй разделы в
`TRANSLATION_NOTES.md`.
